# نشر صفحة سياسة الخصوصية (GitHub Pages)

الملف `docs/privacy.html` جاهز بالكامل — يحتاج فقط تفعيل GitHub Pages
مرة واحدة على مستودعك ليصبح متاحاً كرابط حقيقي.

## الخطوات (مرة واحدة فقط)

1. افتح مستودعك: `https://github.com/abw3laa/halati_app`
2. اذهب إلى **Settings** (أعلى يمين المستودع)
3. من القائمة الجانبية، اختر **Pages**
4. تحت **Build and deployment → Source**، اختر **Deploy from a branch**
5. تحت **Branch**، اختر `main` والمجلد `/docs`، ثم اضغط **Save**
6. انتظر دقيقة إلى دقيقتين، ثم ستظهر في نفس الصفحة رسالة خضراء تحتوي
   الرابط النهائي، وسيكون بهذا الشكل بالضبط:

```
https://abw3laa.github.io/halati_app/privacy.html
```

هذا هو نفس الرابط المضبوط مسبقاً في `lib/utils/app_links.dart`
(`AppLinks.privacyPolicyUrl`) — فلا حاجة لتعديل أي كود، فقط فعّل GitHub
Pages بالخطوات أعلاه.

## صفحة تنزيل التطبيق (اختياري لكن موصى به)

بما أن `AppLinks.appDownloadPageUrl` يشير أيضاً لنفس الموقع
(`https://abw3laa.github.io/halati_app/`)، يمكنك لاحقاً إضافة ملف
`docs/index.html` بسيط يحتوي زر تنزيل APK مباشر (رابط MediaFire) بدل ما
يفتح المستخدم صفحة مستودع GitHub الخام عند الضغط على "مشاركة التطبيق".
أخبرني إن أردت أن أجهّز لك هذه الصفحة أيضاً بنفس الأسلوب.

## تحديث المحتوى لاحقاً

أي تعديل تحتاجه مستقبلاً على سياسة الخصوصية: عدّل مباشرة
`docs/privacy.html` (أو `PRIVACY_POLICY.md` كمصدر أصلي بصيغة أسهل
للتعديل ثم انسخ التعديل يدوياً إلى ملف الـ HTML)، وادفع (`push`) —
GitHub Pages يُحدّث الصفحة المنشورة تلقائياً خلال دقائق.
