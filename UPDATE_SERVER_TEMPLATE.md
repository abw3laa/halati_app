# تحديث حالاتي التلقائي — دليل الإعداد (خارج Google Play)

التطبيق الآن يفحص تلقائياً عند كل تشغيل (وبحد أقصى مرة كل 6 ساعات) ملف
JSON صغيراً تستضيفه أنت، ويقارن رقم الإصدار فيه مع الإصدار المثبّت لدى
المستخدم. إن وُجد إصدار أحدث، تظهر نافذة تعرض ملاحظات التحديث وتتيح
التنزيل والتثبيت مباشرة من داخل التطبيق.

**مهم:** MediaFire ممتاز لاستضافة ملف الـ APK نفسه (رابط تنزيل مباشر
كبير الحجم)، لكنه غير مناسب لاستضافة ملف JSON صغير يحتاج التطبيق قراءته
برمجياً بسرعة. لذلك سنستخدم مكانين مختلفين:

| الملف | أين يُستضاف | لماذا |
|---|---|---|
| `update.json` (صغير، نصي) | GitHub (مجاني) | رابط ثابت، سريع، لا يتطلب تسجيل دخول لقراءته |
| `halati-X.X.X.apk` (كبير) | MediaFire (أو أي مكان تريده) | مناسب للملفات الكبيرة، رابط تنزيل مباشر |

## الخطوة 1: استضافة `update.json` على GitHub

1. أنشئ حساب GitHub مجاني إن لم يكن لديك واحد: https://github.com/signup
2. أنشئ مستودع (Repository) جديد **عام (Public)** باسم مثلاً `halati-updates`
3. داخل المستودع، أنشئ ملفاً باسم `update.json` بهذا المحتوى بالضبط
   (عدّل القيم لاحقاً مع كل إصدار جديد):

```json
{
  "latestVersion": "2.1.0",
  "latestBuildNumber": 2,
  "apkUrl": "ضع هنا رابط MediaFire المباشر لملف الـ APK",
  "minRequiredBuildNumber": null,
  "releaseNotes": {
    "ar": "تحديث تلقائي جديد + إصلاح مشاكل الحفظ",
    "en": "New auto-update system + save bug fixes",
    "tr": "Yeni otomatik güncelleme sistemi + kayıt hata düzeltmeleri"
  }
}
```

### شرح كل حقل:

- **`latestVersion`**: رقم الإصدار الظاهر للمستخدم (يطابق `version` في
  `pubspec.yaml` قبل علامة `+`، مثال: `2.1.0`)
- **`latestBuildNumber`**: **الأهم فعلياً للمقارنة** — رقم صحيح يزيد بمقدار
  1 مع كل إصدار جديد (يطابق الرقم بعد `+` في `pubspec.yaml`، مثال:
  `version: 2.1.0+2` يعني `latestBuildNumber: 2`)
- **`apkUrl`**: رابط MediaFire **المباشر** (وليس رابط صفحة المشاركة —
  انظر الخطوة 3 لكيفية الحصول عليه)
- **`minRequiredBuildNumber`**: اتركه `null` عادة. إن أردت لاحقاً إجبار
  المستخدمين على تحديث ضروري (مثلاً بسبب مشكلة أمنية)، ضع فيه رقم أقل
  إصدار مسموح بالاستمرار بدونه — أي بناء أقدم من هذا الرقم تظهر له نافذة
  تحديث **إجبارية** لا يمكن إغلاقها
- **`releaseNotes`**: نص قصير يظهر للمستخدم يشرح ما الجديد، بثلاث لغات

4. بعد إنشاء الملف، اضغط زر **Raw** أعلى يمين الملف في GitHub — هذا هو
   الرابط الذي يجب وضعه في التطبيق. سيكون شكله مثل:

```
https://raw.githubusercontent.com/اسم_المستخدم/halati-updates/main/update.json
```

## الخطوة 2: وضع رابط GitHub داخل كود التطبيق

افتح الملف:
```
lib/services/update_service.dart
```

وابحث عن السطر:
```dart
static const String manifestUrl =
    'https://raw.githubusercontent.com/YOUR_GITHUB_USERNAME/halati-updates/main/update.json';
```

واستبدل `YOUR_GITHUB_USERNAME` باسم حسابك الفعلي على GitHub، ثم أعد بناء
التطبيق (`flutter build apk --release`) ووزّعه على مستخدميك — هذا الرابط
سيبقى ثابتاً في كل الإصدارات القادمة، فلن تحتاج لتغييره مرة أخرى؛ فقط
تُحدّث محتوى `update.json` نفسه في كل مرة.

## الخطوة 3: رفع ملف الـ APK والحصول على رابط MediaFire مباشر

1. ابنِ نسخة الإصدار: `flutter build apk --release`
2. الملف الناتج يكون في:
   ```
   build/app/outputs/flutter-apk/app-release.apk
   ```
3. ارفعه على MediaFire كالمعتاد
4. بعد الرفع، MediaFire يعطيك رابط **صفحة مشاركة** وليس رابط تنزيل مباشر
   (يبدو مثل `https://www.mediafire.com/file/xxxxx/halati.apk/file`) —
   هذا **لن يعمل** مباشرة كرابط تنزيل تلقائي
5. للحصول على الرابط المباشر:
   - افتح رابط الصفحة في المتصفح
   - اضغط بزر الفأرة الأيمن على زر **Download** الأخضر → **نسخ عنوان
     الرابط (Copy link address)**
   - هذا هو الرابط الذي يبدأ عادة بـ
     `https://download.mediafire.com/...` — ضعه في حقل `apkUrl` داخل
     `update.json`

> ⚠️ **ملاحظة مهمة:** روابط MediaFire المباشرة قد تنتهي صلاحيتها أو
> تتغير أحياناً. إن توقف التحديث التلقائي عن العمل لمستخدميك، أول شيء
> تتحقق منه هو أن رابط `apkUrl` في `update.json` ما زال يعمل (افتحه في
> متصفح خاص/تخفٍ للتأكد أنه يبدأ التنزيل مباشرة بدون صفحة وسيطة).

### بدائل أكثر ثباتاً من MediaFire (اختيارية)

إن أردت رابط تنزيل مباشر لا يتغير أبداً، هذان الخياران مجانيان وأبسط من
MediaFire على المدى الطويل:

- **GitHub Releases**: ارفع ملف الـ APK كـ "Release" في نفس مستودع
  `halati-updates` — يعطيك رابطاً دائماً بصيغة
  `https://github.com/اسمك/halati-updates/releases/download/v2.1.0/halati.apk`
  ولا ينتهي أبداً طالما لم تحذف الإصدار
- **Firebase Hosting** (مجاني حتى حد سخي): استضافة ثابتة بروابط لا تنتهي

## الخطوة 4: كيف يعمل كل هذا داخل التطبيق (لفهمك فقط، لا حاجة لتعديل شيء)

- عند فتح التطبيق، وبحد أقصى مرة كل 6 ساعات، يُرسِل طلباً صامتاً لقراءة
  `update.json`
- إن كان `latestBuildNumber` في الملف أكبر من رقم البناء المثبّت فعلياً
  على جهاز المستخدم، تظهر نافذة "تحديث جديد متاح"
- عند ضغط "تنزيل الآن": يُنزَّل ملف الـ APK من `apkUrl` مع عرض نسبة
  التقدّم، ثم يُطلب من المستخدم إذن "تثبيت من مصادر غير معروفة" (مرة واحدة
  فقط لكل جهاز)، ثم يُفتح مثبّت أندرويد تلقائياً
- من شاشة "الإعدادات"، يوجد أيضاً زر **"التحقق من التحديثات"** يدوي في
  أي وقت، بدون انتظار الـ 6 ساعات

## قائمة تحقق سريعة عند إصدار تحديث جديد لاحقاً

1. ✏️ ارفع `version: X.X.X+N` في `pubspec.yaml` (زِد رقم `N` دائماً)
2. 🏗️ `flutter build apk --release`
3. ☁️ ارفع الـ APK الجديد إلى MediaFire (أو GitHub Releases) واحصل على
   الرابط المباشر
4. 📝 عدّل `update.json` على GitHub: حدّث `latestVersion`،
   `latestBuildNumber`، `apkUrl`، و `releaseNotes`
5. ✅ انتهى — أي مستخدم يفتح التطبيق خلال الساعات القادمة سيرى نافذة
   التحديث تلقائياً
